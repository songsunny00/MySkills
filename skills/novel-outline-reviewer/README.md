# novel-outline-reviewer

网文小说大纲审查 skill，适用于大纲审查、过稿检查、章节质量评估。

---

## 适用场景

| 场景 | 说明 |
|------|------|
| 前三章投稿审查 | 对标主流平台编辑初筛标准，检查开篇钩子、节奏、人设反差、合规项，给出能否直接投稿的结论 |
| 日常章节质量审查 | 从逻辑自洽、人物一致性、世界观、情节连贯性、伏笔、情感共鸣、文笔节奏、内容合规、读者高光体验九个维度全面评估 |
| 内容合规自查 | 快速核查政治、暴力、色情、心理导向、版权六条红线，每条给出明确 PASS/FAIL |

---

## 触发方式

以下任意描述均可激活：

- `审查前三章` / `前三章过稿检查` / `开篇钩子检查`
- `检查这一章` / `review this chapter`
- `人物设定一致吗` / `情节有没有问题`
- `内容合规检查` / `check content safety`
- `故事节奏怎么样` / `improve story flow`

---

## 执行流程

```
输入章节内容
      ↓
审查第1-3章？
  ├─ 是 → 【前置门控】前三章过稿适配审核
  │         ├─ READY        → 继续九维审查
  │         ├─ NEEDS REVISION → 提示问题，可选继续
  │         └─ NOT READY   → 停止，提示用户修改后再审
  └─ 否 → 直接进入九维审查
                ↓
          D1–D9 九维审查
                ↓
          输出报告 + 优先修复清单
```

---

## 前置门控：前三章过稿适配审核

> 仅在审查第 1–3 章时运行，不通过则暂停后续审查，提示用户修改。

| 检查项 | 核心标准 |
|--------|---------|
| 开篇生死线 | 前三行必须出现冲突/危机/悬念，禁止慢热开篇 |
| 钩子密度 | 每 500 字至少一个悬念/钩子 |
| 情绪曲线 | 虐 → 爽 → 虐 → 大爽循环，有明显情绪起伏 |
| 人设反差 | 主角具备可记忆的性格反差，非扁平脸谱 |
| 节奏水分检测 | 无水文凑字，无冗余场景 |
| 核心梗架构 | 普通人 + 异常事件 + 紧迫生死困境 |
| 章节结尾钩子 | 每章末留追更钩，不以平静结局收尾 |
| 投稿合规排查 | 无真实地名、无血腥分尸、无未成年恋爱 |

**总评结论：**
- `READY` — 可直接投稿
- `NEEDS REVISION` — 建议修改后投稿
- `NOT READY` — 必须修改，暂停后续审查

## 九维审查（D1–D9）

> 在门控通过或用户确认后运行，覆盖章节全方位质量。

| 维度 | 检查重点 |
|------|---------|
| D1 逻辑自洽性 | 每个行为是否有动机，时间线/细节是否前后一致 |
| D2 人物设定一致性 | 对话风格、情绪反应、成长弧是否符合角色卡 |
| D3 世界观设定一致性 | 地理空间、时代背景、经济水平、场所规则是否匹配 |
| D4 情节连贯性 | 章节衔接、支线逻辑、场景节奏双功能 |
| D5 伏笔与回收 | 维护伏笔账本，追踪每条线索的埋入与回收状态 |
| D6 感染力与情感共鸣 | Show-don't-tell，五感覆盖，对话自然度 |
| D7 语言节奏与意象 | 句式变化、核心意象一致性、重要场景篇幅 |
| D8 内容红线合规 | 政治/暴力/色情/心理导向/真实性/版权，每项 PASS/FAIL |
| D9 读者高光体验 | 名场面/高光时刻/爽点/撒糖/群像/温馨的密度检查 |

---

## 输出格式

每个维度输出：

```
## [维度名称]
STATUS: PASS / NEEDS ATTENTION / CRITICAL

Issues found:
- [第X章, 场景Y]: [具体问题] → 建议修改方向: [可操作方向]

Strengths worth preserving:
- [值得保留的亮点]
```

结尾附 **优先修复清单**（最多 5 条，按读者体验影响排序）：
门控 NOT READY 项 → D8 CRITICAL → D9 CRITICAL → 其他维度

## 文件依赖

审查前自动并行加载以下上下文（如存在）：

```
characters/*.md      # 角色卡：性格、台词风格、情感基线、成长弧
plot/outline.md      # 大纲：章节钩子、时间线、情节节拍
worldbuilding/world.md  # 世界观：场景规则、文化背景
```

> 前三章门控审查无需以上文件，直接对正文运行。

## 核心原则

- **具体胜于模糊** — 精确到章节、场景、具体台词，不说"人物感觉不对"
- **每条建议可执行** — 指向具体行/段，给出修改方向而非重写
- **保护作者声音** — 提供方向性建议，不强制替换作者文风
- **共鸣优先于技术** — 打动读者的故事可以容忍细微逻辑瑕疵
