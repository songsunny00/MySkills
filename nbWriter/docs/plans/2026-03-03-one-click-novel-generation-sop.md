# 一键生成小说 SOP（标准操作流程）

**文档版本**: 1.0
**创建日期**: 2026-03-03
**目标**: 定义中长篇小说（50,000-80,000字）的自动化生成流程

---

## 一、系统架构概述

### 1.1 核心定位
智能小说创作辅助系统，支持生成50,000-80,000字的中长篇小说。采用"一键启动，分阶段确认"模式，既能快速生成，又保证用户对创作过程的掌控。

### 1.2 双模式设计
- **快速模式**: 用户只需提供类型/主题，系统自动完成所有创作决策
- **详细模式**: 用户可提供完整的人物设定、情节大纲、世界观等详细信息

### 1.3 输出结构
生成完整的创作项目，包含：
- 项目元数据（类型、字数、创作时间等）
- 世界观设定文档
- 人物档案（主角、配角、反派等）
- 完整大纲（章节级）
- 章节内容（Markdown格式）
- 导出文件（TXT、EPUB等多格式）

### 1.4 可扩展性
- 类型模板可配置，支持添加新的小说类型
- 每种类型有独立的生成策略和风格指南
- 支持自定义提示词模板

---

## 二、完整生成流程（7个核心阶段）

### 阶段1：类型分析与参数设定
**目标**: 确定生成基础参数

**操作步骤**:
1. 用户选择小说类型（或系统推荐）
2. 确定模式（快速/详细）
3. 设定目标字数（默认50,000-80,000）
4. 加载对应类型的生成模板

**输出**: 项目配置文件

---

### 阶段2：世界观构建
**目标**: 建立小说的世界观基础

**操作步骤**:
- **快速模式**: 根据类型自动生成世界观框架
- **详细模式**: 用户输入或系统辅助创建
- 包含内容：
  - 时代背景
  - 地理设定
  - 力量体系
  - 社会结构
  - 文化特色

**确认点**: ✋ 用户审核世界观设定

**输出**: `worldbuilding/world.md`

---

### 阶段3：人物设定
**目标**: 创建完整的人物体系

**操作步骤**:
1. 生成主角设定
2. 生成主要配角（3-5个）
3. 生成反派角色
4. 建立人物关系网络

**每个角色包含**:
- 基本信息（姓名、年龄、外貌）
- 性格特征
- 背景故事
- 能力/特长
- 成长弧线
- 与其他角色的关系

**确认点**: ✋ 用户审核人物设定

**输出**: `characters/*.md`

---

### 阶段4：情节设计
**目标**: 规划小说的情节结构

**操作步骤**:
1. 确定核心冲突和主线
2. 设计2-3条副线
3. 规划情感线、成长线
4. 设置关键转折点
5. 设计高潮和结局

**输出内容**:
- 主线情节框架
- 副线情节要点
- 关键情节节点
- 冲突升级路径

**确认点**: ✋ 用户审核情节框架

**输出**: `plot/*.md`

---

### 阶段5：大纲生成
**目标**: 生成详细的章节大纲

**操作步骤**:
1. 根据目标字数计算章节数（通常30-40章）
2. 为每章分配情节点
3. 计算每章字数配额（平均1,500-2,500字/章）
4. 生成详细章节大纲（每章200-300字概要）
5. 标注关键章节：
   - 开篇章节
   - 转折章节
   - 高潮章节
   - 结局章节

**确认点**: ✋ 用户审核完整大纲

**输出**: `plot/outline.md`

---

### 阶段6：智能章节生成
**目标**: 生成完整的章节内容

**生成策略**: 关键章节优先
1. **第1章**（开篇）- 建立基调和世界观
2. **高潮章节**（通常在70-80%位置）- 确定故事走向
3. **结局章节** - 确定收尾方式
4. **关键转折章节**（2-3章）- 重要情节节点
5. **其余章节** - 批量生成（每批5-10章）

**每批生成后**:
- 系统暂停
- 用户可以：
  - ✅ 查看已生成内容
  - ⚙️ 调整后续生成参数
  - 🔄 重新生成不满意的章节
  - ✏️ 直接编辑内容

**确认点**: ✋ 每批生成后用户确认

**输出**: `chapters/chapter-*.md`

---

### 阶段7：润色与导出
**目标**: 完成最终处理和多格式导出

**操作步骤**:
1. **一致性检查**:
   - 人物设定一致性
   - 情节逻辑连贯性
   - 世界观设定统一性
2. **语言润色**:
   - 风格统一
   - 语言流畅度优化
3. **格式处理**:
   - 生成TXT版本
   - 生成Markdown完整版
   - 生成EPUB电子书
4. **项目打包**:
   - 整理所有文档
   - 生成创作报告

**确认点**: ✋ 用户最终审核

**输出**: `exports/*`

## 三、用户控制与编辑能力

### 3.1 实时编辑能力
- 任何阶段的内容都可以直接编辑
- 修改会自动同步到后续生成中
- 支持版本历史，可回退到之前的版本

### 3.2 重新生成选项
- **单章重生成**: 保持上下文连贯性
- **批量重生成**: 重新生成指定范围的章节
- **局部重生成**: 只重写某个场景或段落
- **参数调整**: 可调整生成参数后重新生成

### 3.3 灵活插入
- 在任意位置插入新章节
- 系统自动调整后续章节的衔接
- 重新计算字数分配

### 3.4 参数调整
每个阶段都可以调整生成参数：
- **写作风格**: 简洁/华丽/写实等
- **情节节奏**: 快/中/慢
- **对话比例**: 多对话/多描写/平衡
- **描写详细度**: 简略/适中/详细

### 3.5 分支创作
- 支持创建分支版本
- 可以尝试不同的情节走向
- 对比不同版本后选择保留

## 四、技术实现要点

### 4.1 文件组织结构
```
project-name/
├── metadata.json              # 项目元数据
├── config/
│   ├── genre-template.json    # 类型模板
│   └── generation-params.json # 生成参数
├── worldbuilding/
│   ├── world.md              # 世界观设定
│   └── power-system.md       # 力量体系等
├── characters/
│   ├── protagonist.md        # 主角档案
│   ├── supporting.md         # 配角档案
│   └── relationships.md      # 人物关系
├── plot/
│   ├── outline.md            # 完整大纲
│   ├── main-plot.md          # 主线情节
│   └── subplots.md           # 副线情节
├── chapters/
│   ├── chapter-001.md        # 各章节内容
│   ├── chapter-002.md
│   └── ...
├── exports/
│   ├── novel.txt             # 纯文本版
│   ├── novel.epub            # 电子书版
│   └── novel-full.md         # 完整Markdown
└── history/
    └── versions/             # 版本历史
```

### 4.2 关键技术点

**上下文管理**
- 维护全局上下文（人物、设定、已发生事件）
- 确保生成内容的一致性
- 跨章节的情节连贯性检查

**分段生成**
- 避免单次生成过长
- 采用流式输出提升体验
- 支持中断和恢复

**缓存机制**
- 已确认的内容缓存
- 避免重复生成
- 加速重新生成流程

**提示词工程**
- 每个阶段使用专门优化的提示词模板
- 类型特定的风格指南
- 动态提示词组装

## 五、标准操作流程

### 5.1 启动阶段（1-2分钟）
1. 用户点击"一键生成小说"
2. 选择模式（快速/详细）
3. 选择或输入类型
4. **快速模式**: 输入主题/简介
5. **详细模式**: 填写详细表单
6. 系统显示预估时间和流程图

### 5.2 生成阶段（分步确认）
每个阶段完成后：
1. 系统通知用户查看
2. 展示生成内容
3. 提供操作选项：
   - ✅ 确认继续
   - ✏️ 编辑后继续
   - 🔄 重新生成
   - ⚙️ 调整参数
4. 用户确认后进入下一阶段

### 5.3 章节生成阶段（最耗时）
- 显示实时进度：当前章节/总章节数
- 已生成章节可随时预览
- 支持暂停/继续
- 每批生成后暂停确认

### 5.4 完成阶段
- 展示完整项目结构
- 提供导出选项
- 生成创作报告：
  - 总字数统计
  - 各章节字数
  - 人物出场次数
  - 情节分布图
- 保存项目，支持后续继续编辑

## 六、时间预估

### 各阶段预估时间
- **阶段1-2**: 世界观构建 - 2-3分钟
- **阶段3**: 人物设定 - 2-3分钟
- **阶段4**: 情节设计 - 2-3分钟
- **阶段5**: 大纲生成 - 3-5分钟
- **阶段6**: 章节生成（30-40章）- 30-50分钟
- **阶段7**: 润色与导出 - 3-5分钟

### 总计时间
- **最快**: 约40分钟（快速模式，无重新生成）
- **平均**: 约60分钟（包含审核和少量调整）
- **最长**: 约90分钟（详细模式，多次重新生成）
